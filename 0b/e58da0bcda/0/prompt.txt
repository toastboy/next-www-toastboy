2. Key Technical Concepts:
   - **Mantine `Transition` component**: Used internally by `Button` (when `loading` prop cycles true→false) and by `Select`/`Popover` (when dropdowns open/close). Schedules a `setTimeout` for exit animation. If this timer fires after the test environment (happy-dom) tears down, `window` is undefined → crash.
   - **`vi.useFakeTimers({ shouldAdvanceTime: true })`**: Intercepts `setTimeout` calls as fake timers while still advancing with real wall-clock time. Allows `waitFor` polling and `userEvent` interactions to work normally. `vi.clearAllTimers()` in `afterEach` cancels any pending Mantine animation timers before environment teardown.
   - **Plain `vi.useFakeTimers()` (without `shouldAdvanceTime`)**: Does NOT work here — `waitFor`'s internal retry polling uses `setTimeout` which won't fire without explicit timer advancement, causing tests to hang and timeout.
   - **`userEvent.setup({ advanceTimers: ... })`**: Not needed with `shouldAdvanceTime: true`; plain `userEvent.setup()` works.
   - **Mantine `Select` vs `NativeSelect`**: `NativeSelect` wraps a plain HTML `<select>` (no animations). `Select` uses `Popover` + `Transition` internally (opens/closes with exit timers). With fake timers in place, `Select` is safe to use.
   - **`Select` with Mantine `useForm`**: Must use explicit `value={form.values.field}` and `onChange={(value) => form.setFieldValue('field', ...)}` because `Select`'s `onChange` signature is `(value: string | null) => void`, incompatible with event-based `form.getInputProps()` onChange.
   - **Test interaction pattern for `Select`**: `user.selectOptions()` only works on native `<select>`. Mantine `Select` requires: `await user.click(screen.getByRole('combobox', { name: 'Label' }))` then `await user.click(await screen.findByRole('option', { name: 'Option Label', hidden: true }))`.
   - **happy-dom environment**: Test environment for component tests (not jsdom). Tears down after all tests in a file complete; any still-pending Node.js timers then crash with `window is not defined`.

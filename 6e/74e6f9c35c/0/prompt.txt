findElement only matches when el.type is a function. This will miss common wrapped component types like React.memo / forwardRef (where el.type can be an object) and will also miss vi.fn mocks unless a displayName is set (function name may be 'mockConstructor'). Consider extending matching logic to handle object component types (e.g. check (el.type as any).displayName, (el.type as any).render?.name / .type?.name) or clearly documenting this limitation so tests don’t silently fail to locate elements.

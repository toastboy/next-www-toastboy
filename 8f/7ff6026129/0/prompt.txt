Overriding both pre and code can cause fenced code blocks (<pre><code className=\"language-*\">…) to be processed twice. Depending on MDX’s element mapping order, the pre override may receive a Mantine <Code> child (from the code override) rather than a native <code> element, which can lead to unexpected markup and make it easier to lose language-class/semantics over time. Consider handling fenced blocks exclusively in pre (and making code only style inline code), e.g., have code skip when className indicates a fenced block (language-*) or return a plain code element in that case.

The extra-email path now avoids adding empty normalised addresses, but the accountEmail path earlier in this function still unconditionally adds normalizeEmail(player.accountEmail) when player.accountEmail is truthy (e.g. whitespace-only). This can insert '' into recipients, making recipients.size > 0 and causing an invalid bcc send. Apply the same if (normalizedEmail) recipients.add(normalizedEmail); guard for accountEmail as well.
